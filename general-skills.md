# General Skills — Superpowers для Claude Code / Cursor

> Набор общих AI-агент скиллов от Anthropic (плагин **superpowers**). Не Rezolve-специфичные — универсальные процессы для работы с кодом и задачами.
>
> Установка: плагин ставится один раз и добавляет скиллы в Claude Code. Вызов — через Skill tool автоматически (когда подходит контекст) или вручную через `/superpowers:<name>`.

---

## Process (как подходить к задаче)

| Скилл | Когда вызвать |
|---|---|
| `superpowers:brainstorming` | **Перед** любой творческой работой — новая фича, новый компонент, изменение поведения. Прежде чем писать код. |
| `superpowers:writing-plans` | Когда есть спека / требования и впереди многошаговая задача — до того, как трогать код. |
| `superpowers:executing-plans` | Когда есть готовый имплементационный план и нужно выполнять его в отдельной сессии с чекпойнтами ревью. |
| `superpowers:subagent-driven-development` | Когда план содержит независимые задачи, которые можно делегировать subagent'ам в текущей сессии. |
| `superpowers:finishing-a-development-branch` | Когда имплементация готова, тесты прошли, и нужно решить как интегрировать (merge / PR / cleanup). |

## Code & Review

| Скилл | Когда вызвать |
|---|---|
| `superpowers:test-driven-development` | При написании любой фичи или багфикса — до кода. |
| `superpowers:requesting-code-review` | При завершении задачи или фичи, до мерджа — проверить что результат соответствует требованиям. |
| `superpowers:receiving-code-review` | При получении ревью — особенно если фидбек кажется неясным или технически спорным. Требует rigor, не performative agreement. |
| `superpowers:verification-before-completion` | **Перед** заявлением «работа готова / починено / тесты прошли» — запустить проверки и подтвердить выводом. Evidence before assertions. |

## Debugging

| Скилл | Когда вызвать |
|---|---|
| `superpowers:systematic-debugging` | При любом баге, упавшем тесте, неожиданном поведении — **до** того, как предлагать фикс. |

## Workflow & Infrastructure

| Скилл | Когда вызвать |
|---|---|
| `superpowers:using-superpowers` | Авто — устанавливает как находить и использовать остальные скиллы. Подгружается на старте каждой сессии. |
| `superpowers:using-git-worktrees` | Перед стартом фичи, которая требует изоляции от текущего workspace, или перед выполнением имплементационного плана. |
| `superpowers:dispatching-parallel-agents` | Когда есть 2+ независимых задач, которые можно делать параллельно без общего state. |

## Meta

| Скилл | Когда вызвать |
|---|---|
| `superpowers:writing-skills` | При создании, редактировании или валидации собственных скиллов перед деплоем. |

## Установка

Superpowers — это plugin для Claude Code. Установка раздаётся централизованно по команде — уточни у @Olya Shtalberg, как получить.

После установки скиллы доступны автоматически — Claude сам решает, когда их вызывать. Можно также явно набрать `/<skill-name>` в Claude Code.

## Принцип использования

Если скилл подходит даже на 1% — он должен быть вызван. Не рационализируй «это слишком просто» / «и так понятно» — скиллы не «оверкилл», они дисциплина.
