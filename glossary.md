## Roles & Abbreviations

| Аббревиатура | Расшифровка |
|---|---|
| TPM | Technical Project Manager |
| ITSM | IT Service Management |
| LLM | Large Language Model |
| PM | Project Manager |
| AML Engineer | Applied Machine Learning Engineer — ML-инженер на применении: деплой, обучение и поддержка моделей в реальных системах |
| GPU | Graphics Processing Unit — видеокарта, используется для обучения и запуска ML-моделей. В Rezolve — железо, на котором крутятся модели в Model Garden |
| VM | Virtual Machine — виртуальный компьютер на серверах Rezolve. Инженеры (напр. Mohan) работают не локально, а подключаются к удалённой машине с нужными GPU и доступами |
| SSH | Secure Shell — способ подключиться к удалённой машине (VM) через терминал. Открываешь консоль → вводишь команду → работаешь как будто сидишь за тем компьютером, хотя он в дата-центре |
| SRE | Site Reliability Engineering — команда, отвечающая за деплой и стабильность систем в production. В Rezolve: @Hrvoje (кластеры, prod env) и @Kristijan Belosevic (доступы) принимают модели от ML-команды и выкладывают их |
| PRD | Product Requirements Document — документ с описанием фичи или продукта: зачем нужна, для кого, требования, метрики успеха. Отвечает на *что* и *зачем*, а не *как* |
| Open-weight | Модель, чьи **веса** (числа внутри нейросети, результат обучения) опубликованы и доступны для скачивания. Можно запустить у себя, дообучить, встроить в продукт. Примеры: Qwen3 Omni, Gemma 4, Phi-3. Важно: open-weight ≠ open-source — код обучения и данные могут оставаться закрытыми |
| Open Source (OSS) | В широком смысле — открытый код. В контексте ML в Rezolve используется как синоним open-weight: модели, которые команда скачивает, деплоит на своих GPU и дообучает |
| Проприетарная модель | Модель с закрытыми весами, архитектурой и данными обучения. Доступна только через API, платно за каждый запрос. Примеры: GPT-4, Claude, Gemini. В Rezolve используются как оркестраторы для генерации эталонных диалогов и тестирования |

**Как читать названия Brainpowa / Brain:**

| Паттерн | Что означает | Аналогия |
|---|---|---|
| **Brainpowa** | Название модели — как "Claude" у Anthropic | Anthropic : Claude = Rezolve : Brainpowa |

| **Brain [X]** | SDK, платформенный компонент или продукт поверх LLM | Brain Cortex = SDK/оркестрация; Brain Checkout = продукт |

## Technical Terms

> Алфавитный сводный список Rezolve-специфичных и общих ML-терминов. Сводится из накопленных «Новые термины (авто)» секций.

| Термин | Определение |
|---|---|
| Add to Cart Behavior | Один из четырёх критериев eval-модели — оценка корректности добавления товара в корзину (правильный товар, правильный размер и т.д.). |
| Aion | Внешнее агентство-партнёр Rezolve. Поставляли данные/артефакты для evaluation (Preethi валидирует датасеты от Aion). Сотрудничество завершается — Vladimir закрывает контракт |
| Asking Clarification When Needed | Один из четырёх критериев eval-модели — оценка того, уточняет ли агент недостающие параметры (размер, длина и т.д.) в нужный момент. |
| ASR (Automatic Speech Recognition) | Модели распознавания речи — преобразование аудио в текст (Speech-to-Text). |
| Assert | Явно заданный критерий прохождения теста. В тестировании ПО — это проверка: «ожидаемый результат совпадает с фактическим». В контексте LLM: поскольку вывод модели нестабилен и не детерминирован, Assert — это не строгое равенство, а критерий (например: «ответ содержит приветствие», «tool X вызван», «запрос вне скопа — отклонён»). Матрица тестовых конфигураций без Assert — это список экспериментов, а не тест-план. |
| Aura | Компонент / система внутри Brain Cortex (упомянут Зубейром в контексте багфиксов перед демо). Точное назначение требуется уточнить |
| Azure ML Platform | Облачная платформа Microsoft, используемая Mohan для запуска, остановки и мониторинга GPU-инстансов для деплоя моделей в Model Garden. |
